Редактор правил фильтрации.


Рис 1. Окно редактора правил фильтрации.

Правила фильтрации позволяют отсеять те или иные устройства в процессе сканирования, не затрачивая ресурсов на их обработку.

Все правила фильтрации проверяются на стадии первичного запроса к устройству. Если хотя бы одно из правил сработало, обработка устройства прекращается, и в колонке состояния появится пометка Filtered.

Также правила фильтрации можно изменить, редактируя файл filter.txt.

• Область поиска

В какой области данных искать строку.

- Response body - тело ответа HTTP (содержимое страницы).
- Server name - имя сервера (заголовок Server).
- Realm name - приглашение входа (заголовок WWW-Authenticate: realm).

• Правило проверки

Здесь можно выбрать, в каком случае правило должно срабатывать:

- equal - строка из области данных совпадает с искомой строкой.
- not equal - строка из области данных не совпадает с искомой строкой.
- contains - строка из области данных содержит в себе искомую строку.
- doesn't contain - строка из области данных не содержит в себе искомую строку.

• Искомая строка

Будет выполнен поиск данной строки в заданной области данных.

Чтобы подобрать необходимый набор <область, правило, строка> для конкретного устройства или множества устройств, вам потребуется изучить исходный HTTP ответ с заголовками от устройства.

• Кнопки редактирования

- [Add rule] - добавляет новое правило в конец списка (если заполнены все поля).
- [Save rule] - сохраняет ранее выбранное правило.

Примечание: Не получится добавить новое правило, пока не будет сохранено ранее выбранное.

• Список правил фильтрации

Содержит правила фильтрации, которые будут по очереди проверяться обработчиком устройств.

Выбор существующего правила для редактирования осуществляется двойным нажатием левой кнопки мыши по строке списка.

Удаление выбранного правила из списка осуществляется нажатием кнопки [Del] на клавиатуре.

• Кнопки закрытия окна

Кнопка [OK] сохраняет правила фильтрации и закрывает окно, а кнопка [Cancel] закрывает окно без сохранения изменений.
